Project 1:Scholarly Documents Parse and Search


TASK 1: Parse Metadata & Text form PDFs

Extract metadata, such as title, authors and pages from 1000 papers in PDF format. The machine learning software Grobid is recommended for this task. It can be used to extract fulltext & metadata info from scholarly documents and the results are saved in xml files.
hints
*1000 PDFs(2.61G) can be downloaded here.
*Here is the Grobid Tutorial or you can search grobid on GitHub for more information by yourself.

TASK 2: Index & Search the Parsed Metadata from Grobid

Build the inverted index with the information you have got in the Task 1. The index should includes at least the following fields: title, authors, date, affiliation, address and fulltext.
Write codes to search different fields with queries, e.g. search the word 'computer' in the field 'title'.
hints
*The java-based tool Lucene is recommended to build & search the index.

TASK 3: Build a Simple Web Search Engine with the Index

Implement a simple web search interface to show your search result. The small search engine should at least provide the users with a list of results after a query is submitted and more functions can be added into the website as you like.
hints
*The back-end of the website can be Implemented with IIS+PHP or Tomcat+J2EE+JSP.
*The front-end can be realized by HTML5+CSS+JS. Also, there are many popular front-end framework, such as bootstrap and Foundation.
*For more website construction tutorial, you can refer to w3cshool.
